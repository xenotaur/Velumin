# EV-0003: Modern WebGPU-First `wgpu` Upgrade Verification

## Summary
- Upgraded the browser renderer from `wgpu 0.16` with the `webgl` compatibility feature to `wgpu 29.0.3` using the crate's default WebGPU-first feature set.
- Updated the Rust renderer setup for the modern `wgpu` API while preserving the existing white-line smoke scene.
- Updated crate metadata to declare `rust-version = "1.87"` because `wgpu 29.0.3` requires Rust 1.87 or newer.

## Commands Verified
- From `webgpu_vector_lib/`: `cargo check --target wasm32-unknown-unknown`
- From `webgpu_vector_lib/`: `wasm-pack build --target web`
- From `webgpu_vector_lib/web/`: `npx vite build`
- From `webgpu_vector_lib/web/`: `npm run baseline`

## Result
- `cargo check --target wasm32-unknown-unknown` passed.
- `wasm-pack build --target web` passed and produced `webgpu_vector_lib/pkg`.
- `npx vite build` passed and produced `webgpu_vector_lib/web/dist`.
- `npm run baseline` passed.
- The only remaining Rust warning is the existing wasm-bindgen async-constructor deprecation; no `wgpu` migration errors remain.
