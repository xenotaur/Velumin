# EV-0004: Platform Boundary and Capability Handling Verification

## Summary
- Split the browser-facing `WebGPU` adapter from private renderer state in `webgpu_vector_lib/src/lib.rs`.
- The web adapter now owns browser responsibilities: canvas lookup, `navigator.gpu` detection, DPR backing-store sizing, and resize-triggered render calls.
- The renderer state now owns `wgpu` responsibilities: device, queue, surface configuration, pipeline, surface reconfiguration, and render pass execution.
- Added a page-level status overlay in `webgpu_vector_lib/web/index.html` for unsupported-browser, no-adapter, capability, and render errors.
