# EV-0005: Vector Primitive Tessellation Verification

## Summary
- Added minimal Rust vector command types for `Line` and `Polyline` with width, color, and intensity style data.
- Replaced shader-generated smoke geometry with CPU-generated thick-line triangles.
- Added a vertex format and GPU vertex buffer upload path in the renderer.
- Updated `line.wgsl` to read vertex position and color attributes instead of using `vertex_index` to synthesize geometry.

## Commands Verified
- From `webgpu_vector_lib/`: `cargo test`
- From `webgpu_vector_lib/`: `cargo check --target wasm32-unknown-unknown`
- From `webgpu_vector_lib/web/`: `npm run baseline`

## Result
- `cargo test` passed with 3 tessellation tests.
- `cargo check --target wasm32-unknown-unknown` passed.
- `npm run baseline` passed, including `wasm-pack build --target web` and `vite build`.
- The only remaining Rust warning is the existing wasm-bindgen async-constructor deprecation.

## Tessellation Coverage
- One `Line` command produces 6 vertices, representing two triangles.
- A 3-point `Polyline` produces 12 vertices, representing two thick-line segments.
- Zero-length lines are skipped.
