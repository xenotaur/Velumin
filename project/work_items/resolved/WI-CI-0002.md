# WI-CI-0002: Add GitHub Actions Validation

## Objective
- Add a GitHub Actions workflow that uses the same validation contract as local development.

## Scope
- Add `.github/workflows/validate.yml`.
- Use read-only repository permissions.
- Set up Node 24 with npm caching keyed to `webgpu_vector_lib/web/package-lock.json`.
- Run `scripts/develop` and `scripts/validate`.
- Keep workflow logic focused on setup and repository script execution.

## Evidence
- DP-0004: `project/design/proposals/adopted/DP-0004-script-first-validation-workflow.md`
- Verification evidence: `project/evidence/EV-0007.md`

## Acceptance Criteria
- GitHub Actions calls `scripts/validate` rather than duplicating validation logic.
- CI uses `npm ci` through `scripts/develop`.
- CI uses the checked-in Rust toolchain and WASM target.
- Workflow permissions are limited to `contents: read`.
- Local validation evidence records the workflow syntax and intended CI command path.

## Status
- Done: `.github/workflows/validate.yml` runs `scripts/develop` and `scripts/validate` with read-only repository permissions, Node 24 setup, and npm lockfile caching.
