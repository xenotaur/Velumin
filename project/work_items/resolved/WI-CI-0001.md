# WI-CI-0001: Add Script-First Local Validation

## Objective
- Add the top-level local validation contract for Rust/WASM/Vite development.

## Scope
- Add `rust-toolchain.toml` with Rust 1.87, `rustfmt`, `clippy`, and the `wasm32-unknown-unknown` target.
- Add thin Bash scripts for development setup, version reporting, formatting, linting, tests, baseline builds, and full validation.
- Make `scripts/validate` the canonical local validation command.
- Fix the existing wasm-bindgen async-constructor deprecation so Clippy can run with `-D warnings`.
- Update README to document the new validation entrypoint while preserving browser demo instructions.

## Evidence
- DP-0004: `project/design/proposals/adopted/DP-0004-script-first-validation-workflow.md`
- Verification evidence: `project/evidence/EV-0007.md`

## Acceptance Criteria
- `scripts/version` reports Rust, Cargo, rustfmt, Clippy, wasm-pack, Node, npm, and Vite when available.
- `scripts/format --check` passes.
- `scripts/lint` passes with `-D warnings`.
- `scripts/test` passes.
- `scripts/baseline` passes.
- `scripts/validate` runs the full local validation sequence from the repository root.
- README identifies `scripts/validate` as the canonical local validation command.
